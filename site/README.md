# casim.net — Astro resume site

Static single-page resume site replacing the Flutter Web portfolio (which
moves to <https://flutter.casim.net>, linked in the footer).

## Commands

```sh
npm install
npm run dev      # local dev server
npm run build    # generates public/og.png, then builds to dist/
npm run preview  # serve the production build locally
```

## Content

All content lives in [`src/data/resume.ts`](src/data/resume.ts) as typed
objects — edit there, nothing else. **This repo is public: never add a phone
number or other PII to this file.**

## Architecture notes

- Plain CSS with custom properties (`src/styles/global.css`) — single amber
  accent `#E8A33D` (darkened to `#96610B` for light-theme text to keep
  WCAG AA), 4/8/12/16/24/32/48px spacing scale, 8px card / 6px chip radius.
  Verify token contrast with `node scripts/check-contrast.mjs` (also runs
  in CI).
- Typography: self-hosted Inter variable (body/headings) + JetBrains Mono
  (chips), `font-display: swap`. `public/fonts/*.woff2` are latin+Turkish
  subsets generated by `scripts/subset-fonts.mjs` from the full files in
  `assets/fonts/` — re-run it after upgrading fonts, and commit the output.
- Client JS: the theme toggle, the pre-paint theme init, and the chat widget
  are inlined into the HTML by Astro — no external JS files. The chat widget
  (`src/components/ChatWidget.astro`) renders only when
  `PUBLIC_CHAT_ENDPOINT` is set (see `../worker`); analytics loads only when
  the Umami vars are set (see `../analytics`). Copy `.env.example` to `.env`.
- `scripts/generate-og.mjs` renders the 1200x630 `public/og.png` from
  `resume.ts` at build time (satori + resvg, fonts in `assets/fonts/`).
  It runs automatically via the `prebuild` script.
- JSON-LD (Person + EmployeeRole work history), OpenGraph/Twitter meta, and
  sitemap are wired in `src/layouts/BaseLayout.astro` and
  `astro.config.mjs` (`site: https://casim.net`).

## Deploying to Firebase Hosting

The Astro site is not yet wired into CI. To deploy it instead of the
Flutter build, point the hosting config at `astro/dist`:

```jsonc
// firebase.json (repo root)
"hosting": { "public": "astro/dist", ... }
```

and replace the Flutter build steps in
`.github/workflows/firebase-hosting-merge.yml` with:

```yaml
- uses: actions/setup-node@v4
  with: { node-version: 20 }
- run: npm ci
  working-directory: astro
- run: npm run build
  working-directory: astro
```

The COOP/COEP headers and the SPA rewrite in `firebase.json` exist for the
Flutter/WASM app; the static Astro site doesn't need them (keep the security
headers if the domain still serves the Flutter app anywhere).

## Windows dev machine gotcha

The user-level `~/.npmrc` on this machine sets `os = "linux"`, which makes
npm skip win32 native binaries (e.g. `@resvg/resvg-js-win32-x64-msvc`).
If og-image generation fails with `Cannot find module ...win32-x64-msvc`,
run `npm install` with the override:

```powershell
$env:npm_config_os='win32'; npm install
```
